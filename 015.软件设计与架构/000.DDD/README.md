# DDD(领域驱动设计)
&nbsp;&nbsp;什么是DDD？
- 一种思维方式,关注点从工具回归问题域本身。
- 一组优先任务：以业务为先导
- 一种软件开发方法：系统拆解和集成方法、建模方法、分层架构、实现工具集

战略设计 & 战术设计 （从整体到局部）
## 战略设计
&nbsp;&nbsp;总体设计思路： 分治
### 子域划分:确定逻辑边界
1. 核心域: 重点关注，重点资源投入
2. 支撑域: 专注于业务的某个方面
3. 通用域: 用于整个业务系统
### 限界上下文: 确定物理边界
1. 边界内概念含义统一
2. 不同边界内概念的关注点不同
3. 包含领域模型&系统实现
